---
title: "Busca de Veiculos"
description: "Consulte disponibilidade e precos em tempo real"
---

A busca de veiculos e **assincrona**: voce inicia a busca e depois consulta os resultados por polling. Isso acontece porque o GDS consulta multiplos fornecedores em paralelo, e cada um pode levar alguns segundos para responder.

## Endpoints

| Metodo | Endpoint | Descricao |
|--------|----------|-----------|
| `POST` | `/search` | Iniciar busca de veiculos |
| `GET` | `/search/{search_id}/results` | Consultar resultados (polling) |

---

## Como funciona o fluxo

```
1. POST /search  →  recebe search_id (status: pending)
        |
        | (aguarde 2-5 segundos)
        v
2. GET /search/{id}/results  →  status: pending | complete | failed
        |
        | (repita enquanto status = pending)
        v
3. Quando status = complete  →  lista de veiculos disponivel
```

Recomendamos polling a cada **2 segundos**, com timeout maximo de **30 segundos**.

---

## Iniciar busca

```bash
curl -X POST https://gds.fretatech.com.br/api/partner/v1/search \
  -H "Authorization: Bearer SEU_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "origin": {
      "description": "Aeroporto de Guarulhos, Guarulhos - SP",
      "place_id": "ChIJx7...",
      "lat": -23.4356,
      "lng": -46.4731
    },
    "destination": {
      "description": "Av. Paulista, 1000, São Paulo - SP",
      "place_id": "ChIJy8...",
      "lat": -23.5614,
      "lng": -46.6565
    },
    "trip_date": "2026/04/15",
    "trip_time": "10:00",
    "trip_type": "onewaytrip",
    "passengers": 40
  }'
```

### Campos da requisicao

| Campo | Tipo | Obrigatorio | Descricao |
|-------|------|-------------|-----------|
| `origin.description` | string | Sim | Descricao textual da origem |
| `origin.place_id` | string | Sim | Place ID do Google Maps |
| `origin.lat` | number | Sim | Latitude da origem |
| `origin.lng` | number | Sim | Longitude da origem |
| `destination.description` | string | Sim | Descricao textual do destino |
| `destination.place_id` | string | Sim | Place ID do Google Maps |
| `destination.lat` | number | Sim | Latitude do destino |
| `destination.lng` | number | Sim | Longitude do destino |
| `trip_date` | string | Sim | Data da viagem no formato `YYYY/MM/DD` |
| `trip_time` | string | Sim | Horario da viagem no formato `HH:MM` |
| `trip_type` | string | Sim | Tipo da viagem (veja tabela abaixo) |
| `passengers` | integer | Sim | Numero de passageiros |

### Tipos de viagem

| Valor | Descricao |
|-------|-----------|
| `onewaytrip` | Ida simples |
| `roundtrip` | Ida e volta |
| `hourtrip` | Por hora (com franquia) |
| `customtrip` | Roteiro personalizado (multiplos trechos) |

### Resposta

```json
{
  "data": {
    "search_id": "srch_01jk2m3n4p5q6r7s",
    "status": "pending"
  }
}
```

## Consultar resultados (polling)

```bash
curl -X GET "https://gds.fretatech.com.br/api/partner/v1/search/srch_01jk2m3n4p5q6r7s/results" \
  -H "Authorization: Bearer SEU_TOKEN"
```

### Valores de status

| Status | Descricao |
|--------|-----------|
| `pending` | Busca em andamento — continue o polling |
| `complete` | Busca concluida — resultados disponiveis |
| `failed` | Erro na busca — tente novamente |

### Resposta quando status = pending

```json
{
  "data": {
    "search_id": "srch_01jk2m3n4p5q6r7s",
    "status": "pending",
    "vehicles": []
  }
}
```

### Resposta quando status = complete

```json
{
  "data": {
    "search_id": "srch_01jk2m3n4p5q6r7s",
    "status": "complete",
    "vehicles": [
      {
        "id": "veh_abc123",
        "name": "Ônibus Executivo 50 lugares",
        "capacity": 50,
        "type": "bus",
        "price_cents": 180000,
        "tenant_id": "tenant_xyz789",
        "tenant_name": "Viação Exemplo"
      },
      {
        "id": "veh_def456",
        "name": "Van Executiva 16 lugares",
        "capacity": 16,
        "type": "van",
        "price_cents": 85000,
        "tenant_id": "tenant_abc123",
        "tenant_name": "Transporte Premium Ltda"
      }
    ]
  }
}
```

### Estrutura do veiculo

| Campo | Tipo | Descricao |
|-------|------|-----------|
| `id` | string | Identificador do veiculo na busca (use para criar o orcamento) |
| `name` | string | Nome/descricao do veiculo |
| `capacity` | integer | Capacidade de passageiros |
| `type` | string | Tipo do veiculo (`bus`, `van`, `minibus`) |
| `price_cents` | integer | Preco em centavos (valor do fornecedor, sem margem) |
| `tenant_id` | string | Identificador do fornecedor |
| `tenant_name` | string | Nome do fornecedor |

<Note>
  O campo `price_cents` representa o valor bruto do fornecedor. A margem configurada no GDS sera aplicada ao criar o orcamento. Veja [Orcamentos](/partner/quotes) para mais detalhes.
</Note>

## Exemplo de implementacao do polling

```javascript
async function waitForSearchResults(searchId, token) {
  const maxAttempts = 15; // 30 segundos com intervalo de 2s
  let attempts = 0;

  while (attempts < maxAttempts) {
    const response = await fetch(
      `https://gds.fretatech.com.br/api/partner/v1/search/${searchId}/results`,
      { headers: { Authorization: `Bearer ${token}` } }
    );

    const { data } = await response.json();

    if (data.status === 'complete') {
      return data.vehicles;
    }

    if (data.status === 'failed') {
      throw new Error('Busca falhou. Tente novamente.');
    }

    // Ainda pendente — aguarda 2 segundos antes de tentar novamente
    await new Promise(resolve => setTimeout(resolve, 2000));
    attempts++;
  }

  throw new Error('Timeout: busca nao concluiu em 30 segundos.');
}
```
